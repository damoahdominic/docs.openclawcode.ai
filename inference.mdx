## Request Flow

When you use AI features in OCCode, here's what happens:

```
OCCode Editor
    │
    │  POST /v1/chat/completions
    │  Authorization: Bearer <jwt>
    │  { model, messages, stream }
    ▼
occ.mba.sh (API Gateway)
    │
    │  1. Validates JWT token
    │  2. Checks credit balance (402 if empty)
    │  3. Forwards to inference backend
    │  4. Deducts credits based on token usage
    ▼
LiteLLM (inference.mba.sh)
    │
    │  Routes to best available model
    ▼
OpenRouter → AI Model
```

## Endpoint

The inference endpoint is **OpenAI-compatible**:

```
POST https://occ.mba.sh/v1/chat/completions
```

### Request

```bash
curl https://occ.mba.sh/v1/chat/completions \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "moltpilot",
    "messages": [
      {"role": "user", "content": "Write a Python hello world"}
    ],
    "stream": true
  }'
```

### Response (non-streaming)

```json
{
  "id": "chatcmpl-abc123",
  "object": "chat.completion",
  "choices": [{
    "message": {
      "role": "assistant",
      "content": "print('Hello, World!')"
    }
  }],
  "usage": {
    "prompt_tokens": 12,
    "completion_tokens": 8,
    "total_tokens": 20
  }
}
```

### Streaming

When `stream: true`, responses are sent as **Server-Sent Events (SSE)**. The final chunk includes usage data for credit deduction.

## Credit Deduction

Credits are deducted **per request** based on actual token usage:

```
cost = (tokens_in × $0.0000005 + tokens_out × $0.0000015) × markup
```

| Factor | Value |
|--------|-------|
| Input token rate | $0.50 per 1M tokens |
| Output token rate | $1.50 per 1M tokens |
| Markup | Configurable (server-side) |

- **Streaming requests**: Usage is extracted from the final SSE chunk
- **Non-streaming requests**: Usage is read from the JSON response body
- **Deduction is atomic**: Uses `GREATEST(0, balance - cost)` to prevent negative balances

## Rate Limits

| Limit | Value |
|-------|-------|
| Inference requests | 60 per minute per user |
| Auth endpoints | Rate limited per IP |

## Error Responses

| Status | Meaning |
|--------|---------|
| `402` | Credit balance depleted. Top up at the [dashboard](https://occ.mba.sh/dashboard). |
| `401` | Invalid or expired JWT token. Re-authenticate. |
| `429` | Rate limit exceeded. Wait and retry. |
| `502` | Upstream inference provider error. Retry after a moment. |
