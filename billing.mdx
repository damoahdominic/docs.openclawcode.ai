## How Credits Work

OCCode uses a **prepaid credit system**. You load credits into your account, and they're deducted as you use AI features.

<Steps>
  <Step title="Sign up">
    Every new account starts with **$5.00 in free credits**.
  </Step>
  <Step title="Use AI">
    Each AI request (chat, inline edit, code generation) deducts a small amount based on tokens used.
  </Step>
  <Step title="Top up">
    When your balance runs low, buy more credits from the dashboard.
  </Step>
</Steps>

## How Much Do Things Cost?

AI usage is metered by **tokens** — the units of text that language models process.

| Activity | Approximate Cost | Credits Used |
|----------|-----------------|--------------|
| Short question ("fix this bug") | ~500 tokens | ~$0.001 |
| Code generation (function) | ~2,000 tokens | ~$0.003 |
| Long conversation (10 messages) | ~10,000 tokens | ~$0.01 |
| Large refactor | ~50,000 tokens | ~$0.05 |

With **$5 in free credits**, you can make roughly **5,000+ short requests** or **100+ substantial code generation sessions**.

## Buying Credits

### From the Dashboard

1. Go to [occ.mba.sh/dashboard](https://occ.mba.sh/dashboard)
2. Navigate to **Credits** in the sidebar
3. Click a credit tier ($5 / $10 / $20 / $50)
4. Complete payment via **Stripe Checkout**
5. Credits are added to your account instantly

### Payment Methods

Stripe supports:
- Credit and debit cards (Visa, Mastercard, Amex)
- Apple Pay and Google Pay
- Bank transfers (select regions)

## Checking Your Balance

### In the Dashboard

The [dashboard](https://occ.mba.sh/dashboard) shows your current balance, total spent, and request count on the **Overview** page.

### Via API

```bash
curl https://occ.mba.sh/api/v1/me \
  -H "Authorization: Bearer YOUR_TOKEN"
```

The `balance_usd` field in the response shows your current credit balance.

### In OCCode

Your balance is passed to the editor during authentication via the deep link:

```
occode://openclaw.home/auth?token=<jwt>&balance=<amount>
```

## What Happens When Credits Run Out?

When your balance hits **$0.00**:

- AI requests return a **402 error** with a link to top up
- The editor shows a prompt to add credits
- All other editor features (editing, git, terminal) continue to work normally
- **No data is lost** — you just can't make AI requests until you top up

## Viewing Usage History

The dashboard **Usage** page shows every AI request with:

- Model used
- Input and output tokens
- Cost deducted
- Timestamp

The **Credits** page shows your purchase history with amounts and dates.
